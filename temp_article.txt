CC Switch下载与DeepSeek配置完全指南
一、CC Switch是什么？
CC Switch是一款跨平台的开源桌面应用，用于统一管理Claude Code、Codex、Gemini CLI等AI编程工具的模型配置。简单来说，它像一个"AI模型切换器"——你把各个模型的API Key和接口地址存进去，点一下就能切换，再也不用手动改配置文件了。

CC Switch内置了50多家供应商的预设配置，选择预设后只需填写API Key即可使用。目前支持管理Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw等主流AI编程工具。

二、下载与安装
CC Switch支持Windows、macOS和Linux三大平台。访问 GitHub Releases（https://github.com/farion1231/cc-switch/releases）即可下载最新版本。

Windows用户提供两种安装方式。

方式一：MSI安装包（推荐）。下载 CC-Switch-v{版本号}-Windows.msi 安装包，双击运行安装程序，按向导完成安装。建议将安装路径从默认的C盘修改至非系统盘（如D盘），避免系统盘空间不足。

方式二：绿色便携版（免安装）。下载 CC-Switch-v{版本号}-Windows-Portable.zip ，解压到纯英文路径（中文路径可能导致数据库初始化失败），直接运行 CC-Switch.exe 即可。

如果遇到Windows SmartScreen拦截，点击"更多信息"→"仍要运行"即可正常启动。

macOS用户同样提供两种方式。

方式一：Homebrew安装（推荐）。执行以下命令：

brew tap farion1231/ccswitch
brew install --cask cc-switch

这种方式支持一键安装和自动更新。

方式二：手动安装。下载 CC-Switch-v{版本号}-macOS.dmg 镜像文件，双击打开，将 CC Switch.app 拖入「应用程序」文件夹。首次启动时若提示"无法验证开发者"，前往「系统设置」→「隐私与安全性」，点击「仍要打开」完成授权。也可下载 .zip 压缩包，解压后直接运行。

Linux用户根据发行版选择对应的安装包。Debian/Ubuntu下载 .deb 包安装，Fedora/RHEL下载 .rpm 包安装。通用方案是下载 .AppImage 文件，添加执行权限后直接运行：

chmod +x CC-Switch-v{版本号}-Linux.AppImage
./CC-Switch-v{版本号}-Linux.AppImage

三、准备工作：获取DeepSeek API Key
在配置CC Switch之前，你需要先有一个DeepSeek的API Key。

第一步，访问 DeepSeek 平台官网，注册并登录账号。

第二步，完成实名认证（国内用户可用支付宝或微信充值，按量计费）。

第三步，进入API密钥管理页面，点击「新建API密钥」。

第四步，复制并妥善保存生成的API Key（通常以 sk- 开头）。

DeepSeek官方文档标明的OpenAI兼容接口地址为 https://api.deepseek.com ，Chat接口路径为 /chat/completions 。

四、在CC Switch中配置DeepSeek
以下以配置 Codex CLI 使用 DeepSeek 为例（配置 Claude Code 的操作类似）。

第一步，确认CC Switch版本。确保CC Switch版本为 3.16.0及以上，该版本起内置了DeepSeek预设，无需手动配置协议参数。

第二步，切换到对应应用标签。打开CC Switch，在顶部应用切换器中，根据你要配置的目标工具选择对应的标签——例如配置Codex就切换到「Codex」标签。

第三步，添加DeepSeek供应商。点击右上角的「+」按钮，打开添加供应商面板。在预设列表中选择「DeepSeek」。选择预设后，接口地址、默认模型等字段会自动填充。在API Key字段中粘贴你之前获取的DeepSeek API Key。点击「保存」或「添加」完成配置。

第四步，启动本地路由（针对Codex用户）。如果你配置的是Codex CLI，还需要额外开启本地路由功能。点击CC Switch右上角的「设置」图标，进入「路由」页面。开启「本地路由总开关」，本地代理服务会在 127.0.0.1:15721 上启动。在「路由启用」中打开 Codex选项。

这里解释一下本地路由的作用。Codex原生只支持OpenAI的Responses API，而DeepSeek使用的是Chat Completions API，两者协议不兼容。CC Switch在中间完成协议转换，让Codex无需修改就能调用DeepSeek。

第五步，验证配置。配置完成后，在对应CLI工具中发起一次请求，检查是否能正常返回结果。如果遇到问题，可以检查API Key是否正确、网络是否能正常访问DeepSeek的接口地址、路由开关是否已开启（Codex用户）。

五、配置完成后能做什么？
配置好之后，你可以一键切换模型，在CC Switch主界面或系统托盘中秒级切换不同供应商。你还可以添加更多模型，以后想接入其他模型如通义千问、Kimi等，同样的操作只需30秒。CC Switch还支持集中管理配置，所有API Key和模型配置统一存放在CC Switch中，安全且便于管理。

六、注意事项
第一，Node.js环境。CC Switch和Claude Code、Codex等工具都依赖Node.js环境，建议提前安装Node.js 18或更高版本。

第二，CLI工具需提前安装。CC Switch本身是配置管理工具，你需要先安装好对应的CLI工具（如Claude Code或Codex），再通过CC Switch来管理它的模型配置。

第三，切换后需重启终端。在CC Switch中切换配置后，建议重启终端或IDE，让新的环境变量生效。

第四，API Key安全。CC Switch将API Key保存在本地，不会上传到任何服务器。开启本地路由后，Codex的配置中使用的是占位符，真实的Key始终保存在CC Switch中。

第五，关于DeepSeek的计费。DeepSeek采用按量计费模式，API调用会产生费用，建议在使用前了解清楚计费标准，避免意外超额。
