所有文件操作遵循三级分类路径规则，默认使用中文路径，仅在中文路径会导致功能异常时回退到英文路径：

## 路径规则

| 类型 | 默认路径 | 说明 |
|------|----------|------|
| **我制作的项目** | `D:\AI\AI project\` | 代码、脚本、文档、生成的文件 |
| **安装的环境/依赖/工具** | `D:\软件\工具\` | Node、Python、SDK、CLI 工具等 |
| **安装的软件** | `D:\软件\` | 桌面应用程序 |

## 回退规则

仅当**中文路径会导致软件无法正常运行**时，才使用英文路径：
- 环境/工具 → `D:\software\tool\`
- 软件 → `D:\software\`

## 操作原则

1. 安装前先确认目标路径是否存在，不存在则创建
2. 优先选择可自定义安装路径的版本（非 Microsoft Store 版）
3. 操作前主动说明将要使用的路径
4. 中文路径优先，功能不兼容才回退英文

**Why:** 用户希望保持文件系统整洁有序，中文路径优先但不受限于中文路径导致的兼容性问题。

**How to apply:** 每次涉及文件生成、下载、安装操作时，先判断属于哪一级分类，选择合适的目录，必要时询问用户。
